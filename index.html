Chipstart

Spring 2016

Chipstart is a web app meant to help kids learn to write simple code. It used a simulated on-screen microchip that would "control" LEDs or a 7-segment display, and "take input" from toggle switches and/or buttons (all simulated on-screen as well). The idea was that by offering a simple visual medium, controlled by a small number of simple commands and elementary math, coding could be made to seem less daunting and easier to learn.

In the spring of 2017, I submitted to the TechU.Me AppJam (an app competition for high school students), and won a cash prize as well as a scholarship to the University of Ottawa!

The app worked via a JavaScript interpreter (I was not familiar with cloud computing back then) that parsed and executed a syntax of code developed by myself, but very similar in syntax to Python. The language was fully documented such that if read from the beginning, users could learn to write it with no prior programming experience.

The app also featured a small number of "challenges" that the user could complete. The users could write and test their code, and when they were ready, they could have it "evaluated". The app would check that the code did what it was supposed to, and then give feedback based on the results of running it.

To date, I am probably most proud of this project. Unfortunately, it isn't live at the moment, but I hope to recreate it with more features (and a more elegant solution than an interpreter written in JavaScript). The code can be found on a repository on my GitHub.

Turd Sort

Winter 2015

Turd Sort was my answer to BOGO sort. It uses the same principle as bogo sort: randomize the list until it just happens to be in the correct order. Turd Sort improves upon bogo sort by shuffling the list until either the min or max winds up at the front or end of the list. When this happens, Turd Sort will keep it there and shuffle the middle of the list, until there are 0 or 1 items left in the middle.

On average, Turd Sort was way faster than bogo sort, especially at list sizes above 6. Ultimately, though, Turd Sort is useless as a sorting algorithm. I think its purpose is to serve as a reminder: an algorithm may be able to be improved, but sometimes it's better to write a new one altogether.

The Python code can be found on a repository on my GitHub. You can also see graphs of its average runtimes over 5,000 trials, and its average runtimes compared to bogo sort, also over 5,000 trials.
